撤回已push的commit

社区寻找到的解决方案是本地先 reset 到提交前版本，然后放弃掉本不需要的更改，最后直接–force 提交

尝试了一下–force 被 gitlab 拒绝了，估计是管理员设置了不允许强制提交，所以这里记录下另一种解决方案

gitlab 使用 revert 撤销已 push 的 commit

revert 的原理是 gitlab 自动创建一个了新的分支（此分支最后记得手动删除），把撤销后的代码存进新分支，然后再将新分支合并过来
